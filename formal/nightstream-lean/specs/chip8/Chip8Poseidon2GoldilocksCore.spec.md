# CHIP-8 Poseidon2 Goldilocks Core

## Purpose

This owner fixes the exact concrete Poseidon2-over-Goldilocks width-8
permutation used by the CHIP-8 simple-kernel transcript. It closes the last
explicit cryptographic parameter beneath `Chip8Root0Digest` and
`Chip8ChallengeDerivation`.

## Scope

This module defines:

- the exact width-8 initial external-round constants
- the exact width-8 terminal external-round constants
- the exact width-8 internal round constants
- the exact Goldilocks internal diagonal matrix for width `8`
- the exact degree-`7` S-box
- the exact external linear layer using the `MDSMat4`-based width-8
  `mds_light_permutation`
- the exact internal linear layer `matmul_internal(_, MATRIX_DIAG_8_GOLDILOCKS)`
- the exact initial-external, internal, and terminal-external round order
- the concrete width-8 core value exported to `Chip8Root0Digest`

This module does not define:

- transcript packing or labels
- `root0` preimage ownership
- any Stage 1 / Stage 2 / Stage 3 semantic theorem

## Canonical parameter source

The concrete permutation is the deterministic width-8 Goldilocks instance
generated by the live Rust stack from:

- seed `neo_poseidon2_goldilocks_seed___`
- `Poseidon2Goldilocks::<8>::new_from_rng_128`
- `p3-goldilocks 0.4.1`
- `p3-poseidon2 0.4.1`

The Lean owner freezes the resulting constants directly. It does not model the
ChaCha8 RNG or constant-generation pipeline internally.

## Exact round structure

Let `state ∈ F^8`.

### Initial external layer

The initial external layer is:

1. one width-8 `mds_light_permutation` using the `MDSMat4` matrix
2. four external rounds, each:
   - add the round-constant vector coordinatewise
   - apply the degree-`7` S-box coordinatewise
   - apply the width-8 `mds_light_permutation`

### Internal layer

There are exactly `22` internal rounds. Each round:

1. adds one scalar round constant to coordinate `0`
2. applies the degree-`7` S-box to coordinate `0`
3. applies

$$
\mathrm{matmul\_internal}(\_, \mathrm{MATRIX\_DIAG\_8\_GOLDILOCKS})
$$

so that each output coordinate is:

$$
state_i \cdot diag_i + \sum_{j=0}^{7} state_j.
$$

### Terminal external layer

The terminal external layer consists of four further external rounds with the
same coordinatewise `(+ rc)`, degree-`7` S-box, and width-8
`mds_light_permutation`.

## Exported concrete core

The owner exports one exact value:

$$
\mathrm{concreteCore} : \mathrm{Poseidon2Width8Core}
$$

whose `permute` function is this exact concrete permutation.

## Fixed vector data

This owner must additionally expose deterministic expected permutation vectors
for the following canonical inputs:

- all-zero state
- range state `[0,1,2,3,4,5,6,7]`
- all-one state

These are the theorem-owned reference values that the later Rust↔Lean parity
checks consume.
